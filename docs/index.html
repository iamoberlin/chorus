Information Flow

Illumination flows down. Each morning, Seraphim sets the mission context. That context cascades through increasingly frequent execution layers. By the time it reaches Angels, the mission has been translated into moment-to-moment action.

Insight flows up. Angels observe patterns. Principalities synthesize them into research. Virtues turn them into improvements. Cherubim consolidate them into lasting knowledge.

Recursive Self-Improvement

The Virtues choir is why CHORUS matters. Six times per day, it:

  1. Reviews recent performance ‚Äî what worked, what failed, why
  2. Identifies improvement opportunities
  3. Generates modifications ‚Äî config changes, prompt updates, new automations
  4. Assesses risk ‚Äî low-risk changes apply automatically; high-risk get flagged
  5. Logs everything to CHANGELOG.md

The Powers choir validates changes adversarially. Together, they create a tight feedback loop.

This compounds.

Day 1, your agent is baseline. Day 7, it's fixed the obvious issues. Day 30, it's running workflows you never imagined. Day 90, it's unrecognizable from where it started.

Most agents stay frozen. CHORUS agents evolve.

Why Architecture?

Separation of Concerns

Strategic thinking doesn't compete with routine monitoring. Research doesn't interfere with security review. Each function has dedicated resources and appropriate pacing.

Temporal Hierarchy

Mission review shouldn't happen every 30 minutes. Heartbeat checks shouldn't happen once a week. The architecture enforces appropriate rhythm.

Integrated Security

Powers choir runs adversarial checks 8√ó/day. Combined with OpenClaw's core security layer, your agent is defended in depth.

Graceful Scaling

Add domains by extending Principalities. Add capabilities by extending Virtues. The architecture grows without collapsing into chaos.

RSI Protocol

The virtues choir implements recursive self-improvement:

1. ANALYZE
   Read memory files, CHANGELOG.md, recent sessions
   Identify: successes, failures, patterns

2. PROPOSE
   Generate modification (config, prompt, automation)
   Assess risk: low | medium | high

3. APPLY
   Low-risk: implement directly, log to CHANGELOG.md
   Medium-risk: implement, flag for review
   High-risk: write to proposals/, await human approval

4. VALIDATE
   powers choir reviews changes adversarially
   Reverts if issues detected
